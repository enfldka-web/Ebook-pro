# Atlas Product Vision

관련 문서: [Master Roadmap](./MASTER_ROADMAP.md) · [Current Features](./CURRENT_FEATURES.md)

---

## 한 문장 정의

주제나 원천 자료를 입력하면 전자책 제작부터 판매 준비 자료(썸네일·상세페이지)까지 한 흐름에서 완성하도록 돕는 AI 기반 제작 SaaS.

(현재는 SaaS의 서버/과금 인프라 없이, 사용자가 자신의 Anthropic API 키를 브라우저에 입력해 사용하는 클라이언트 전용 프로토타입 단계입니다. "SaaS"는 Phase 4~5에서 도달할 목표 형태이며, 현재 상태를 SaaS로 표현하지 않습니다.)

## 해결하려는 문제

- 전자책 제작 과정 자체가 기획·집필·편집을 아우르며 복잡함
- 기획, 본문 작성, 표지, 썸네일, 상세페이지 제작이 서로 다른 도구/과정으로 분산되어 있음
- 처음 전자책을 판매하는 사용자는 상세페이지·썸네일 같은 "판매 준비 자료"를 어떻게 만들어야 할지 모름
- 여러 챕터, 여러 판매 자료를 만드는 과정에 반복 작업이 많음
- AI가 생성한 결과물의 품질을 사용자 스스로 판단하기 어려움(무엇을 고쳐야 하는지 감이 없음)

## 핵심 사용자

**1차 사용자**
- 전자책 초보 판매자 — 처음 디지털 상품을 만들어 파는 사람

**2차 사용자**
- 1인 지식사업자
- 프리랜서
- 콘텐츠 제작자

**향후 사용자** (Phase 4 이후, 서버/계정 체계가 갖춰진 뒤 확대 고려)
- 교육/컨설팅 사업자
- 여러 명이 함께 작업하는 소규모 팀

## 핵심 가치 제안

- 제작 시간 단축 — 기획부터 판매 자료까지 하나의 워크스페이스에서 진행
- 한 흐름에서 결과물 완성 — 전자책 생성 → 썸네일 → 상세페이지로 자연스럽게 이어짐
- 판매 준비 자료 자동화 — 크몽 등록용 문구, 썸네일, 카드뉴스형 상세페이지를 함께 제공
- 선택·수정·미리보기 — AI가 만든 결과를 그대로 쓰지 않고 사용자가 직접 고를 수 있음(제목 후보, 템플릿, 색상 등)
- 내부 품질 평가와 개선 제안 — 만든 결과물이 어떤 부분에서 약한지 알려줌(Quality Coach, Thumbnail Intelligence)

## 차별점

- 전자책 "본문"만 생성하는 도구가 아니라, 썸네일·상세페이지·등록용 문구까지 하나의 프로젝트 안에서 연결함
- 생성으로 끝나지 않고, 생성 이후 평가와 개선 제안까지 제공함(단, 이는 내부 휴리스틱 평가이며 실제 시장 성과 예측이 아님 — 아래 "성공 지표 후보" 및 각 기능 문서 참고)
- 한국 사용자와 국내 판매 플랫폼(크몽 등)의 관행을 고려해 문구·정책 검사를 설계함

## 제품 원칙

1. 사용자가 클릭해야 하는 횟수를 줄인다 — 필요한 조작은 한 화면 안에서 끝나도록 구성한다.
2. 생성보다 수정 가능성을 중요하게 본다 — AI가 만든 결과를 사용자가 직접 고치거나 다른 안을 고를 수 있어야 한다.
3. 실제 성과를 보장하는 과장 표현을 하지 않는다 — CTR·매출·전환율 등을 예측하거나 보장하는 표현은 UI·문서 어디에도 쓰지 않는다.
4. 검증되지 않은 기능을 완료로 표시하지 않는다 — 실제 브라우저 테스트를 거치지 않은 기능은 "완료"라고 보고하지 않는다.
5. 기존 기능을 깨지 않는 작은 단위로 개발한다 — 새 기능은 기존 기능과 병렬로 추가하고, 회귀 테스트로 기존 동작을 확인한 뒤에만 병합한다.
6. 사용자가 결과를 소유하고 다운로드할 수 있어야 한다 — 생성된 전자책, 썸네일, 판매 문구는 항상 파일로 내려받을 수 있어야 한다.

## 비전과 비전이 아닌 것

**Atlas는 이것입니다**
- 전자책 제작과 판매 준비를 돕는 제작 도구
- AI 생성 결과를 사용자가 검토·수정할 수 있게 구조화한 워크플로우

**Atlas는 이것이 아닙니다**
- Atlas는 판매 준비를 돕는 도구이지만, 수익이나 판매량을 보장하는 서비스가 아닙니다.
- Atlas는 AI가 생성한 결과를 검토 없이 완벽하다고 주장하지 않습니다(품질 점수·개선 제안은 참고용 내부 평가입니다).
- Atlas는 아직 결제·구독·팀 협업을 지원하는 완성된 SaaS가 아닙니다(Phase 4~5 목표).

## 성공 지표 후보

**아직 실제 측정 인프라가 구축되어 있지 않으므로, 아래는 향후 측정을 고려할 후보일 뿐이며 확정된 목표 수치가 아닙니다.**

- 첫 결과물 생성 완료율
- 생성 시작부터 다운로드까지 소요 시간
- 썸네일/상세페이지 수정 횟수
- 프로젝트 저장 및 재방문율
- Trial → Pro 전환율 (과금 체계 도입 이후에만 의미 있음)
- 월간 활성 사용자
- 오류율 / 생성 실패율

이 지표들을 실제로 수집하려면 서버 기반 로깅(Phase 4)이 필요하며, 현재는 어떤 지표도 실측되고 있지 않습니다.
